# narrative-product-launch

**描述**：产品发布预热叙事教练。内嵌叙事元系统，帮助用户把产品卖点/发布节奏/目标用户变成有逻辑、有冲突、有认知、有价值的产品发布预热叙事。

## 🎯 功能

### 输入字段（四层素材）
1. 产品名称/核心卖点：
2. 发布日/预热期长度：
3. 目标用户（画像/兴趣/痛点）：
4. 希望用户最终产生的认知（一句话）：

### 输入校验规则
- **必填字段**：4 个输入字段全部必填，缺任何一项需提示用户补充
- **长度限制**：每个字段 20-500 字，过短需追问细节，过长需提炼核心
- **格式要求**：用户输入为自然语言，AI 自动解析为结构化素材
- **受众画像**（可选）：提供时用于调整叙事角度和语言风格
- **叙事人格**（可选）：不填时根据素材特征自动推荐（ENTP/ENFP/ENFJ）
- **校验失败处理**：字段缺失 → **AI 必须主动追问缺失字段**，直到 4 个字段全部提供；长度不足 → 追问细节；格式错误 → 引导正确格式

### 结构
核心谜底提炼 → 碎片化拆解 → 顿悟点设计 → 探索指引方案 → 奖励机制 → 执行排期表

### 输出
- 碎片地图（5-10 个碎片，含触点/解锁条件/依存关系）
- 顿悟时刻脚本（用户完成拼合时的触发文案/动效）
- 探索指引（用户如何一步步发现，含官方提示力度建议）
- 奖励机制（即时奖励/终极奖励）
- 执行排期表（按时间线排列）

## ⚠️ 常见错误

1. **写成产品说明书**：只罗列功能和参数，没有故事和悬念，缺乏吸引力
2. **写成广告文案**：只有卖点，没有碎片化和探索过程，缺乏参与感
3. **只讲成功**：没有失败和试错过程，像机器人写的
4. **产品过度完美**：没有局限性和适用边界，用户不信任
5. **空喊口号**：没有具体的碎片设计和执行排期

## 🧪 使用示例

### 示例1：碎片地图
用户输入：
```
产品名称/核心卖点：TWS 耳机，主打"超长续航 + 骨传导通话降噪"
发布日/预热期长度：30 天后
目标用户：25-35 岁职场人，经常通勤，需要清晰通话
希望用户最终产生的认知：这副耳机能陪我跑完马拉松，还能清楚接电话。
```

AI输出：
```
【人格推荐】ENTP
理由：素材特征为颠覆性创新、解谜导向、互动设计，ENTP叙事风格最适合。

【核心谜底提炼】
最终用户应自己发现的真相：**"这副耳机续航 12 小时，通话降噪让风声人声分离。"**

【碎片化拆解】（10 个碎片）

| 碎片编号 | 名称 | 内容 | 触点 | 解锁条件 | 依存关系 |
|---------|------|------|------|----------|----------|
| F01 | 倒计时 | 官网首页倒计时，数字不是数字，是"剩余听歌 X 首" | 官网 | 首次访问 | 无 |
| F02 | 马拉松 | 官方微博发布"马拉松路线图"，中途有个小点标注"通话清晰区" | 微博 | 发布后第 3 天 | 无 |
| F03 | 客服彩蛋 | 用户问"续航多久"，回复"听满 7 部指环王" | 客服 | 主动咨询 | 无 |
| F04 | 隐藏动效 | 产品详情页隐藏彩蛋：鼠标滑过耳机图，出现"风太大？我不怕"动效 | 官网 | 浏览详情页 | 无 |
| F05 | 跑步视频 | 抖音放出 15 秒视频：人在跑步机上，边跑边打电话，对方听不到风声 | 抖音 | 发布后第 7 天 | 无 |
| F06 | 用户证言 | "我跑完半马，电话那头说：你那边风很大吗？我说：没有啊。" | 小红书 | 发布后第 10 天 | 需要 F05 |
| F07 | 技术揭秘 | 骨传导通话降噪原理动画（30 秒） | B 站 | 发布后第 14 天 | 无 |
| F08 | 倒计时故事 | 发布前 3 天，每天发布一个"通勤故事"短视频 | 短视频平台 | 发布前 3 天 | 无 |
| F09 | 碎片收集进度条 | 小程序上线"碎片收集"功能，每发现一个碎片点亮一枚图标 | 小程序 | 发布后第 7 天 | 无 |
| F10 | 终极顿悟卡片 | 集齐 9 个碎片后，小程序解锁动画卡："你发现了——它能陪你跑完马拉松，还能清楚接电话。" | 小程序 | 集齐前 9 个碎片 | 需要所有其他碎片 |

【顿悟时刻脚本】
- 触发条件：用户集齐 9 个碎片（F01-F09）
- 仪式感：屏幕飘落碎片，逐渐拼成耳机轮廓，同时播放一段 15 秒的温暖背景音乐
- 可分享物料：自动生成一张个人海报，文案为"我发现了耳机的秘密：[用户输入的通勤故事]"

【探索指引方案】
- 指引强度：中指引
- 官方提示：小程序内设"侦探笔记"板块，前三个碎片用问号图标

【奖励机制】
- 即时奖励：每发现一个碎片，小程序点亮一枚徽章
- 终极奖励：集齐 9 个碎片可生成一张 9 折优惠券

【执行排期表】
- 第 1-7 天：上线 F01、F02、F04（官网 + 微博基础碎片）
- 第 8-14 天：上线 F03、F06（客服彩蛋 + 用户证言）
- 第 15-21 天：上线 F05、F07、F09（跑步视频、技术揭秘、进度条）
- 第 22-28 天（发布前）：上线 F08（倒计时故事）、F10（终极顿悟）
- 第 29 天后：保持所有碎片在线，但不主动提示，作为长期彩蛋
```

## 📋 质量检查清单

### 叙事质量（由 narrative-meta 八刀评估驱动）
- [ ] 有开头抓力？（核心谜底提炼）
- [ ] 有逻辑递进？（碎片→顿悟→指引→奖励→排期）
- [ ] 有真实感？（具体细节锚点：触点/解锁条件/依存关系）
- [ ] 有认知提炼？（产品哲学/用户价值，不是"我们有什么"，而是"用户会发现什么"）
- [ ] 有价值输出？（读者能带走的具体方法/视角/行动）
- [ ] 适合产品发布表达？（与目标用户画像匹配）

### 伦理与合规（由 narrative-meta 驱动）
- [ ] 伦理审查通过？（无事实篡改/无不当省略/无隐私泄露）
- [ ] 去AI化检测通过？（7条规则全部通过）

### 输入校验
- [ ] 4个必填字段全部提供？
- [ ] 每个字段长度在20-500字范围内？
- [ ] 受众画像/叙事人格已确认？
